# 🇩🇪 German Flashcards

A simple, self-hosted web app for learning German vocabulary **in context** —
full sentences tailored to **your career** (DevOps, team lead, design, or anything
you type) and **your CEFR level** — through a guided **lesson → quiz → unlock** flow.

- **First-time onboarding** asks your **career track**, **CEFR level** and **words
  per lesson**, then tailors the deck (career picks which categories you study).
- **Study** a batch of N new words (flip a card to reveal meaning), then take a
  **multiple-choice quiz**: see a word and pick the sentence that uses it correctly.
- Score **≥ 80%** to **unlock the next batch**; below that, restudy and retry.
- **Review mistakes** — words you answer wrong go into a review list; quiz them again
  and a correct answer clears them.
- Each **word** can have **several example sentences**; one is shown **at random**.
- **Career-specific sentences via AI:** the vocabulary (words) is curated by **admins**;
  when a word has no example sentence for *your* career, the app generates a few with an
  **AI provider** (Claude by default, pluggable) and **caches them shared per career**.
- **Private notes:** add your own private note to any word (never shown to other users).
- Browse the deck (admins can add/edit/delete words and sentences).
- **No duplicates:** adding a word that already exists (case-insensitive) merges the
  new sentences into the existing word, and identical sentences are skipped.
- **Grammar section** at `/grammar`: a left sidebar of topics; click one to read its
  rules (Markdown with tables for cases/conjugations). Ships with a **24-topic German
  reference** (`seed_grammar.py`) grouped into Grundlagen, Pronomen, Verben,
  Zeitformen, Präpositionen, Adjektive, Syntax, Alltag — the static source of truth.
- **Accounts to keep your progress** — sign up with an **email + password** (stored
  locally, hashed with PBKDF2) or **Google login** (optional). Guests' progress is
  saved in the browser; signing in syncs it to your account across devices.
- **Daily reminder** — opt-in browser notification ("You have N words to study or
  review") at a time you pick. Toggle it at the top of the Learn tab (asks for
  notification permission). Fires while the app has a tab open on a new day past your
  set time; settings are saved per browser.
- Data is stored locally in **SQLite** (`flashcards.db`).

The vocabulary deck (words + sentences) is **shared by everyone**; your **profile and
learning state are per user**. The lesson/quiz logic runs in the browser, so guest and
signed-in users behave identically — only *where* progress is stored differs.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Run

```bash
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

Then open **http://localhost:8000**.

On first run the database is created and seeded with ~30 starter cards
(categories: `devops`, `incident`, `team-lead`, `meetings`).

> **Phone tip:** with `--host 0.0.0.0` you can open the app from your phone on the
> same Wi-Fi at `http://<your-computer-ip>:8000`.

## Accounts & login

Two ways to sign in (both optional — the app runs fully in **guest mode** with no
setup, progress saved in the browser's `localStorage`):

- **Local accounts** — work out of the box. Click **Sign up**, enter an email +
  password (min 8 chars). Passwords are hashed with **PBKDF2-HMAC-SHA256** (standard
  library, no plaintext stored). No configuration needed.
- **Google login** — optional, requires OAuth credentials (below).

On first sign-in (either method), any progress you built up as a guest is merged
into your account automatically.

### Admins & AI sentence generation

Set these in `.env` (see `.env.example`):

- **`ADMIN_EMAILS`** — comma-separated emails allowed to add/delete words (must be
  signed-in users). Everyone else studies and can trigger AI generation, but can't
  edit the deck. The **Add** tab only appears for admins.
- **AI provider** — career-specific sentences are generated by an AI provider behind a
  pluggable interface (`ai.py`), selected by `AI_PROVIDER` (default `claude`).
  - `ANTHROPIC_API_KEY` — required to enable generation (uses the Anthropic Python SDK).
  - `ANTHROPIC_MODEL` — defaults to `claude-opus-4-8`.
  - Without a key, generation is disabled and the app falls back to existing sentences
    (no errors). Generated sentences are stored **shared, tagged by career**, so the AI
    is called only once per (word, career). Add more providers by registering them in
    `ai.py`'s `PROVIDERS` map.

### Optional: Google login

To enable the "Continue with Google" button:

1. Go to **https://console.cloud.google.com/apis/credentials**
2. **Create credentials → OAuth client ID → Web application**
3. Under *Authorized redirect URIs* add: `http://localhost:8000/auth/callback`
4. Copy `.env.example` to `.env` and fill in the values:
   ```bash
   cp .env.example .env
   # then edit .env: GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, SESSION_SECRET
   ```
   Generate a session secret with:
   ```bash
   python -c "import secrets; print(secrets.token_hex(32))"
   ```
5. Restart the server. A **Sign in with Google** button now appears.

When a guest who already studied signs in for the first time, their browser
progress is merged into their account automatically.

> If `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` are missing, login is simply
> disabled and the app stays in guest mode — nothing breaks.

## How the lesson flow works

1. **Onboarding** (first visit) — pick career track, level, and words-per-lesson.
2. **Study** the next N un-learned words from your track (tap a card to reveal the
   word + translation).
3. **Quiz** — for each word, choose the sentence that uses it correctly (1 right
   answer + 2 distractor sentences from other words).
4. **≥ 80%** marks those words *learned* and unlocks the next N. Below 80%, the
   lesson stays so you can restudy / retry.
5. **Review mistakes** quizzes the words you've missed; getting one right clears it.

Change your career/level/goal anytime via **⚙︎ Settings** on the Learn tab.

## Project layout

| File | Purpose |
| ---- | ------- |
| `app.py` | FastAPI backend: auth, shared deck, per-user profile + learning state |
| `db.py` | SQLite — `users`, `words`, `sentences`, `profiles`, `learning`, `grammar` |
| `seed.py` | Starter words/sentences used to seed an empty database |
| `seed_grammar.py` | Starter grammar topics (Markdown) |
| `static/index.html` | Main UI; runs the lesson/quiz logic client-side |
| `static/grammar.html` | Grammar page (sidebar + Markdown reader) |
| `.env.example` | Template for optional Google OAuth credentials |

## API

| Method & path | Purpose |
| ------------- | ------- |
| `GET /api/words?career=` | Words with career-relevant + generic sentences (`has_career_sentences` flag) |
| `POST /api/words/{id}/generate` | Generate & cache AI sentences for a career: `{career, level, n}` |
| `POST /api/words` *(admin)* | Add a word: `{word, category, sentences:[{sentence_de, sentence_en}]}` |
| `POST /api/words/batch` *(admin)* | Add many words (a JSON array of the above) |
| `POST /api/words/{id}/sentences` *(admin)* | Add a sentence to an existing word |
| `DELETE /api/words/{id}` *(admin)* | Delete a word and all its sentences |
| `DELETE /api/sentences/{id}` *(admin)* | Delete one sentence |
| `GET` / `PUT /api/notes[/{word_id}]` | *(signed in)* private per-user notes on words |
| `GET /api/categories` | Distinct categories |
| `GET` / `PUT /api/profile` | *(signed in)* career / level / daily_goal |
| `GET` / `PUT /api/learning` | *(signed in)* `{learned:[ids], wrong:{id:count}}` |
| `GET /api/grammar` | Grammar topics for the sidebar (titles, no body) |
| `GET /api/grammar/{slug}` | One grammar topic with its Markdown body |
| `POST /api/grammar` | Add/update a topic: `{title, body, category, position}` (upsert by slug) |
| `DELETE /api/grammar/{id}` | Delete a grammar topic |
| `GET /api/me` | Current user + whether Google login is configured |
| `POST /auth/signup` | Create a local account: `{email, password, name?}` |
| `POST /auth/login` | Log in to a local account: `{email, password}` |
| `POST /auth/logout` | End the session |
| `GET /auth/google`, `GET /auth/callback` | Google OAuth flow (if configured) |
| `GET /api/progress` | *(signed in)* `{word_id: {reps, interval, ease, lapses, due}}` |
| `PUT /api/progress/{word_id}` | *(signed in)* upsert progress for one word |

Spaced-repetition scheduling and "what's due" are computed in the browser; guests
persist progress in `localStorage`, signed-in users via `/api/progress`.

Interactive docs: **http://localhost:8000/docs**.

## The database

All data lives in **`flashcards.db`** (SQLite). **Do not delete it** — it caches
AI-generated example sentences that cost tokens/money to produce, plus every user's
accounts, progress, notes and token balances.

Schema changes are handled automatically by **additive migrations** in `db.py`
(`_migrate()` adds new columns in place), so upgrading never requires wiping the DB.

If you ever need a clean slate, **back up first** (`cp flashcards.db flashcards.bak`)
so the generated sentences aren't lost.
